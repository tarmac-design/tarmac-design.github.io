## States

| State | Description |
|-------|-------------|
| Default | Unchecked, ready for interaction |
| Hover | Slight background highlight on container |
| Checked | Filled background with checkmark icon |
| Indeterminate | Dash icon, used for "select all" patterns |
| Focused | 2px blue outline for keyboard navigation |
| Disabled | 40% opacity, non-interactive |
| Error | Red border with error message below |

## Usage Guidelines

<Info>
  Use checkboxes when users can select multiple options. For mutually exclusive choices, use Radio buttons instead.
</Info>

<Warning>
  Avoid using a single checkbox for binary actions — use a Toggle/Switch instead for immediate on/off settings.
</Warning>

---

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Semantic HTML | Use native `<input type="checkbox">` |
| Keyboard | Tab to focus, Space to toggle |
| Focus ring | 2px visible outline, ≥ 3:1 contrast |
| Label | Always associate with `<label>` or `aria-label` |
| Indeterminate | Set via `aria-checked="mixed"` |
| Group | Wrap related checkboxes in `<fieldset>` with `<legend>` |

## Developer Handoff

```tsx
interface CheckboxProps {
  checked?: boolean;
  indeterminate?: boolean;
  disabled?: boolean;
  error?: boolean;
  size?: 'md' | 'sm';
  label?: string;
  description?: string;
  onChange?: (checked: boolean) => void;
}
```

```tsx
<Checkbox label="Accept terms" />
<Checkbox label="Newsletter" description="Receive weekly updates" />
<Checkbox checked indeterminate label="Select all" />
```
