## Usage Guidelines

<Info>
  Use radio buttons when users must select exactly one option from a group. Always provide at least two options.
</Info>

<Warning>
  Never use a single radio button alone — for binary choices, use a Checkbox or Toggle instead.
</Warning>

---

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Semantic HTML | Use native `<input type="radio">` with shared `name` |
| Keyboard | Tab to group, Arrow keys to navigate options |
| Focus ring | 2px visible outline, ≥ 3:1 contrast |
| Label | Always associate with `<label>` or `aria-label` |
| Group | Wrap in `<fieldset>` with `<legend>` |
| ARIA | Use `role="radiogroup"` for custom implementations |

## Developer Handoff

```tsx
interface RadioProps {
  value: string;
  checked?: boolean;
  disabled?: boolean;
  error?: boolean;
  size?: 'md' | 'sm';
  label?: string;
  description?: string;
  onChange?: (value: string) => void;
}

interface RadioGroupProps {
  name: string;
  value?: string;
  direction?: 'horizontal' | 'vertical';
  error?: string;
  children: React.ReactNode;
  onChange?: (value: string) => void;
}
```

```tsx
<RadioGroup name="shipping" value="express" onChange={setValue}>
  <Radio value="standard" label="Standard" description="5-7 business days" />
  <Radio value="express" label="Express" description="2-3 business days" />
  <Radio value="overnight" label="Overnight" description="Next business day" />
</RadioGroup>
```
