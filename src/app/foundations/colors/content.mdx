# Colors

TARMAC's color system is built on a 3-layer architecture — **Core → Semantic → Usage** — that balances brand expression with functional clarity across all Delhivery products.

## Principles

<Info>
  Always use semantic tokens in your UI code rather than raw hex values. Semantic tokens ensure your interfaces adapt correctly to themes and maintain consistent meaning across contexts.
</Info>

- **Purposeful** — Every color has a defined role and communicates meaning
- **Accessible** — All color pairings meet WCAG 2.1 AA contrast requirements (4.5:1 for text, 3:1 for UI elements)
- **Systematic** — A layered architecture that scales from brand to component level

## Color Architecture

TARMAC uses a 3-layer token system:

1. **Core Tokens** — Raw color values (hex codes). These are the palette.
2. **Semantic Tokens** — Purpose-driven aliases (e.g., `color.text.primary`). These describe *what* a color does.
3. **Usage Tokens** — Component-specific mappings (e.g., `button.background.primary`). These describe *where* a color is used.

This layering means you can update the entire visual language by changing core values without touching component code.

## Core Brand Colors

| Token | Name | Hex | Usage |
|-------|------|-----|-------|
| `brand.black` | Black | `#000000` | Primary text, high-contrast elements |
| `brand.white` | White | `#FFFFFF` | Backgrounds, inverse text |
| `brand.red` | DLV Red | `#ED1B36` | Brand accent, primary actions |
| `brand.blue` | Blue | `#2396FB` | Links, informational elements |
| `brand.success` | Success | `#1BA86E` | Positive states, confirmations |
| `brand.warning` | Warning | `#CF9F02` | Caution states, alerts |
| `brand.error` | Error | `#DC143C` | Error states, destructive actions |

## Usage Guidelines

<Info>
  **Do:** Use `color.text.primary` for main body text. Use `color.surface.secondary` for card backgrounds. Use semantic tokens that describe the *purpose* of the color.
</Info>

<Warning>
  **Don't:** Hard-code hex values in component styles. Don't use brand red for non-brand or non-destructive actions. Don't rely on color alone to communicate state — always pair with icons or text.
</Warning>

## Semantic Tokens

Semantic tokens are grouped into four categories:

| Category | Examples | Description |
|----------|----------|-------------|
| **Surface** | `color.surface.primary`, `color.surface.secondary` | Background colors for containers, cards, pages |
| **Text** | `color.text.primary`, `color.text.secondary`, `color.text.disabled` | Foreground colors for text content |
| **Icon** | `color.icon.primary`, `color.icon.secondary` | Colors for iconography |
| **Border** | `color.border.default`, `color.border.focus` | Stroke colors for outlines and dividers |

## Code Example

```css
/* Using TARMAC color tokens as CSS custom properties */
.card {
  background-color: var(--color-surface-primary);
  border: 1px solid var(--color-border-default);
  color: var(--color-text-primary);
}

.card-title {
  color: var(--color-text-primary);
}

.card-description {
  color: var(--color-text-secondary);
}

.button-primary {
  background-color: var(--color-brand-red);
  color: var(--color-text-inverse);
}
```
